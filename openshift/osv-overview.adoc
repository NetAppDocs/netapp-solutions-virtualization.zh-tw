= 了解 Red Hat OpenShift Virtualization
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Red Hat®OpenShift® 虛擬化技術使企業能夠在統一的混合雲端應用程式平台上，無縫遷移、執行及管理傳統虛擬機器 （VM） 與現代工作負載。OpenShift 虛擬化功能以操作員形式提供，包含於任何 OpenShift 訂閱中。此 OpenShift Container Platform 附加元件為虛擬機器與容器化應用程式的整合提供了強大的解決方案。它使用基於核心的虛擬機器 （KVM）——值得信賴的 Linux 核心 Hypervisor，以其強大的效能、可擴展性和穩定性而著稱——在 OpenShift 中以原生物件形式為虛擬機器提供企業級虛擬化。

image:redhat-openshift-044.png["OpenShift 虛擬化"]



[[key-features-of-red-hat-openshift-virtualization]]
== 紅帽 OpenShift 虛擬化的主要特性

**混合工作負載：**遷移並執行具有雲端原生優勢的 VM，使用統一的工具集將其與容器和無伺服器工作負載一起管理。

**逐步現代化：**您可以按照自己的節奏，透過採用容器、實施微服務或整合 AI 來現代化，而不會中斷現有的 VM。

**Kubernetes 整合：** 利用 Kubernetes 自訂資源，為您的 OpenShift 叢集新增虛擬化功能。



[[capabilities-of-red-hat-openshift-virtualization]]
== Red Hat OpenShift Virtualization 的功能

**虛擬機器管理：**建立和管理 Linux 和 Windows 虛擬機器。

**統一工作負載：** 在叢集中同時執行 pod 和 VM 工作負載。

**連接方式：** 透過各種控制台和 CLI 工具存取虛擬機器。

**虛擬機器操作：** 在節點之間匯入、複製和即時遷移虛擬機器。

**資源管理：** 管理連接到虛擬機器的網路介面和儲存磁碟。



[[compatibility]]
== 相容性

雖然 OpenShift Virtualization 在概念上與 VMware vSphere 不同，但它提供了一個專為虛擬化管理員量身打造的現代化平台，並提供熟悉的功能，例如：**即時移轉：** 相當於 vSphere 中的 vMotion。

**儲存管理：** OpenShift 中的儲存類別與 vSphere 中的儲存原則型管理 （SPBM） 保持一致。

**持久性儲存：**持久性磁碟區宣告 (PVC) 和持久性磁碟區 (PV) 對應於 vSphere 中的資料存放區。
