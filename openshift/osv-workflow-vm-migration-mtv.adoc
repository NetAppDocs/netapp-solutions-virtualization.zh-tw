= 將虛擬機器從 VMware 遷移到 Red Hat OpenShift 集群
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
使用 OpenShift 虛擬化移轉工具套件，將虛擬機器從 VMware 移轉至 OpenShift 叢集。此移轉包括安裝虛擬化移轉工具套件 （MTV）、建立來源和目的地供應商、建立移轉計劃，以及執行冷移轉或熱移轉。

.冷遷移
[%collapsible%open]
====
這是預設的移轉類型。來源虛擬機器將在資料複製期間關閉。

====
.暖遷移
[%collapsible%open]
====
在這種類型的遷移中，大部分的資料是在來源虛擬機器 (VM) 運行時的預複製階段複製。然後在切換階段關閉虛擬機器並複製剩餘資料。

====

== 使用虛擬化遷移工具包將虛擬機器從 VMware 遷移到 OpenShift 虛擬化

本節說明如何使用 Migration Toolkit for Virtualization (MTV) 將虛擬機器從 VMware 移轉至在 OpenShift Container Platform 上執行，並使用 Trident 與 NetApp ONTAP 儲存設備整合的 OpenShift Virtualization。

下圖顯示虛擬機器從 VMware 移轉到 Red Hat OpenShift Virtualization 的高階檢視。

[[prerequisites-for-the-sample-migration]]
=== 範例遷移的先決條件



[[on-vmware]]
=== 在 VMware 上

* 安裝了一台執行 RHEL 9.3 的 RHEL 9 虛擬機器，配置如下：
+
** CPU：2個，記憶體：20GB，硬碟：20GB
** 使用者憑證：root 使用者和 admin 使用者


* 虛擬機器準備就緒後，即安裝 PostgreSQL 伺服器。
+
** PostgreSQL 伺服器已啟動並設定為開機時自動啟動。




[source, console]
----
systemctl start postgresql.service
systemctl enable postgresql.service
----
這些命令確保伺服器在移轉後能夠在 OpenShift Virtualization 的虛擬機器中啟動。

* 新增了 2 個資料庫、1 個資料表和 1 列資料。如需在 RHEL 上安裝 PostgreSQL 伺服器以及建立資料庫和資料表項目的說明，請參閱 link:https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/9/html/configuring_and_using_database_servers/using-postgresql_configuring-and-using-database-servers#configuring-postgresql_using-postgresql["這裡"]。



NOTE: 確保啟動 PostgreSQL 伺服器並啟用開機啟動服務。



[[on-openshift-cluster]]
=== 在 OpenShift 叢集上

在安裝 MTV 之前已完成以下安裝：

* OpenShift Cluster 4.17 或更高版本
* 為 iSCSI 啟用的叢集節點上的多路徑（用於 ontap-san 儲存類別）。如果使用 node-prep 標誌安裝Trident 25.02，則可以輕鬆啟用多路徑。您可以參考link:osv-trident-install.html["Trident安裝部分"]了解詳情。
* 安裝所需的後端和儲存類別以及快照類別。請參閱link:osv-trident-install.html["Trident安裝部分"]了解詳情。
* link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.19#Virtualization["OpenShift 虛擬化"]




[[install-mtv]]
=== 安裝 MTV

現在您可以安裝 Migration Toolkit for Virtualization （MTV）。請參閱提供的說明 link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/installing-the-operator["這裡"] 以取得安裝協助。

虛擬化移轉工具套件 (MTV) 使用者介面已整合到 OpenShift Web 控制台中。您可以參考 link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/migrating-vms-web-console#mtv-ui_mtv["這裡"] 開始使用使用者介面執行各種任務。



==== 建立來源供應商

若要將 RHEL 虛擬機器從 VMware 移轉到 OpenShift 虛擬化環境，您需要先建立 VMware 的來源供應商。請參閱link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/migrating-vms-web-console#adding-providers["這裡"]以取得建立來源供應商的說明。

您需要以下內容來建立 VMware 來源提供者：

* vCenter URL 位址
* vCenter 憑證
* vCenter 伺服器指紋
* 儲存庫中的 VDDK 映像


來源提供者建立範例：

image:rh-os-n-use-case-vm-migration-source-provider.png["此圖顯示輸入/輸出對話框或表示書面內容"]


NOTE: 虛擬化遷移工具包 (MTV) 使用 VMware 虛擬磁碟開發工具包 (VDDK) SDK 來加速從 VMware vSphere 傳輸虛擬磁碟。因此，儘管是可選的，但強烈建議創建 VDDK 影像。若要使用此功能，您需要下載 VMware 虛擬磁碟開發工具包 (VDDK)，建立 VDDK 映像，然後將 VDDK 映像推送到您的映像登錄。

請按照提供的說明進行操作link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/prerequisites#creating-vddk-image_mtv["這裡"]建立 VDDK 映像並將其推送到可從 OpenShift 叢集存取的登錄。



==== 建立目的地供應商

主機叢集會自動新增為目的地供應商。



==== 制定移轉計劃

請按照提供的說明進行操作link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/migrating-vms-web-console#creating-migration-plan_mtv["這裡"]建立遷移計劃。

創建計劃時，如果尚未創建，則需要創建以下內容：

* 將來源網路對應到目標網路的網路映射。
* 儲存設備映射用於將來源資料儲存映射到目標儲存設備類別。為此，您可以選擇 ontap-san 儲存設備類別。建立移轉計劃後，計劃狀態應顯示為 *Ready*，此時您應能夠 *Start* 該計劃。


image:rh-os-n-use-case-vm-migration-mtv-plan-ready.png["此圖顯示輸入/輸出對話框或表示書面內容"]



[[perform-cold-migration]]
=== 執行冷遷移

點選「開始」，依照步驟順序完成虛擬機器的移轉。

image:rh-os-n-use-case-vm-migration-mtv-plan-complete.png["此圖顯示輸入/輸出對話框或表示書面內容"]

所有步驟完成後，您可以透過點擊左側導覽功能表「虛擬化」下的「虛擬機器」來查看已移轉的虛擬機器。若要管理虛擬機器，請參閱 link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.22/html/virtualization/managing-vms["OpenShift 虛擬化文件"]。

您可以登入虛擬機器並驗證 PostgreSQL 資料庫的內容。資料庫、表格以及表格中的項目應該與來源 VM 上建立的內容完全一致。



[[perform-warm-migration]]
=== 執行熱遷移

若要執行熱移轉，在建立如上所示的移轉計畫後，您需要編輯計畫設定以變更預設移轉類型。按一下冷移轉旁邊的編輯圖示，然後切換按鈕將其設定為熱移轉。按一下 *Save*。現在按一下 *Start* 以啟動移轉。


NOTE: 在 VMware 中從區塊儲存設備移轉時，請確保已為 OpenShift Virtualization 虛擬機器選擇區塊儲存設備類別。此外，volumeMode 應設定為 Block，存取模式應設定為 ReadWriteMany （RWX），以便稍後執行虛擬機器的即時移轉。

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-001.png["1"]

點選「已完成 1 個虛擬機器中的 0 個」，展開虛擬機器，即可查看移轉進度。

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-002.png["2"]

一段時間後，磁碟傳輸完成，移轉將等待進入切換狀態。DataVolume 處於暫停狀態。返回計劃並按一下 *Cutover* 按鈕。

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-003.png["3"]

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-004.png["4"]

對話方塊顯示目前時間。您可以變更時間以安排稍後進行切換，或按一下 *Set cutover* 立即執行切換。

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-005.png["5"]

幾秒鐘後，當轉換階段開始時，DataVolume 會從「已暫停」狀態變更為 ImportScheduled，再變更為 ImportInProgress 狀態。

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-006.png["6"]

切換階段完成後，DataVolume 狀態變成成功，PVC 即會繫結。

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-007.png["7"]

移轉計畫繼續完成 ImageConversion 階段，最終完成 VirtualMachineCreation 階段。虛擬機器在 OpenShift 虛擬化環境中進入執行中狀態。

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-008.png["8"]
