= 使用 NetApp Console Backup and Recovery Service 備份與還原 VM
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
使用 NetApp Console 備份與恢復 Red Hat OpenShift Virtualization 中的 VM。此程序涵蓋透過 NetApp Console 的 Backup and Recovery Service 建立與管理備份與恢復作業，並使用 Azure Blob Storage Container 作為備份目標。

NetApp Console 提供對本地和雲端環境中儲存設備和資料服務的集中管理。它提供統一的控制、簡易的操作和安全的管理。使用者可以透過使用者介面存取多種資料服務和管理功能，網址為 link:https://console.netapp.com["console.netapp.com"^]。有關 NetApp Console 的完整詳細資訊，請參閱 link:https://docs.netapp.com/us-en/console-family/["NetApp Console 文件"]。

本節重點介紹 NetApp Backup and Recovery 資料服務，適用於 Kubernetes 工作負載，特別是 OpenShift 虛擬機器。NetApp Backup and Recovery Service 可讓您透過單一介面來探索、管理、建立保護原則，並將這些原則與您的 Kubernetes 應用程式和 VM 建立關聯。如需完整指南，請參閱 link:https://docs.netapp.com/us-en/data-services-backup-recovery/br-use-kubernetes-protect-overview.html["NetApp Backup and Recovery 文件"]。



[[backup-and-recovery-workflow]]
== 備份與恢復工作流程

. *請確保您已安裝 Console Agent*
+
請確保在執行 OpenShift 叢集的環境中已部署主控台代理程式。有關安裝主控台代理程式的詳細資訊，請參閱 link:https://docs.netapp.com/us-en/console-setup-admin/concept-agent-install-options.html#agent-console-deploy["NetApp Console Setup 文件"]。

+
.顯示範例
[%collapsible]
====

在本例中，我們選擇本地部署。複製代理程式安裝程式 URL，使用 curl 命令在執行於 vSphere 上的 Ubuntu 機器上下載代理程式安裝程式，然後執行安裝程式以部署 Console 代理程式。

使用代理程式的 IP 位址在 URL 中進行註冊。請參閱link:https://docs.netapp.com/us-en/console-setup-admin/task-install-agent-on-prem-ova.html["文件"]以取得詳細說明。代理程式註冊完成後，您可以在 NetApp Console 中看到它，如下快照所示。

. *將 Azure Blob 儲存 Container 設定為 OpenShift VM 備份與恢復方案的備份目標。*
+
在本例中，我們使用 Azure Blob 儲存體作為備份目標。您也可以使用 AWS S3、Google Cloud Storage、StorageGRID 或 ONTAP 物件儲存作為備份目標。

+
如需更多詳細資訊，請審查 link:https://docs.netapp.com/us-en/data-services-backup-recovery/br-start-setup.html#prepare-the-backup-destination["NetApp Console 文件"]。

+
若要在 NetApp Console 中探索 Azure Blob 儲存 Container 作為備份目標，您需要從 Azure 入口網站取得下列資訊。

+
[listing]
----
Application (Client) ID:
Client Secret:
Directory (tenant) ID:
----
+
若要取得這些值，請在 Azure 入口網站中執行下列操作。

+
.. 建立 Blob 儲存 Container。
.. 建立企業應用程式，並為該應用程式指派 Blob 儲存 Container 所需的存取。取得該應用程式的應用程式 ID、目錄 ID 和用戶端金鑰，以便在 NetApp Console 中將 Azure Blob 儲存 Container 探索為備份目標。
+
.顯示範例

如果您的應用程式/虛擬機器有多個磁碟，且備份失敗並出現以下錯誤，請編輯 `volumegroupsnapshots`和 `volumegroupsnapshotcontents` API 規則，使其包含所示的動詞。

.<strong>顯示修復</strong>
[%collapsible]
=====
image:osv-data-protection-console-043.png["保護每個含有超過 1 個磁碟的虛擬機器時發生錯誤"]

[source, yaml]
----
- apiGroups:
  - groupsnapshot.storage.k8s.io
  resources:
  - volumegroupsnapshots
  verbs:
  - get
  - list
  - watch
  - create
  - delete
  - update
  - patch
- apiGroups:
  - groupsnapshot.storage.k8s.io
  resources:
  - volumegroupsnapshotcontents
  - volumegroupsnapshotcontents/status
  verbs:
  - get
  - list
  - watch
  - create
  - delete
  - update
  - patch
- apiGroups:
  - groupsnapshot.storage.k8s.io
  resources:
  - volumegroupsnapshotclasses
  verbs:
  - get
  - list
  - watch

----
