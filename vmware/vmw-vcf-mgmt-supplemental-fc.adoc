= 使用適用ONTAP tools for VMware vSphere新增基於 FC 的 VMFS 資料儲存庫作為管理域的補充存儲
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
在此用例中，我們概述如何透過光纖通道 (FC) 配置 VMFS 資料儲存作為 VMware Cloud Foundation (VCF) 管理網域的補充儲存。此過程總結了在管理網域上部署ONTAP工具、新增儲存後端以及配置資料儲存庫的步驟。

.開始之前
確保以下組件和配置已到位。

* 具有連接到 FC 交換器的 FC 連接埠的ONTAP儲存系統。
* 使用 FC LIF 建立的 SVM。
* 帶有 FC HBA 的 vSphere 連接到 FC 交換器。
* 在 FC 交換器上設定單一啟動器-目標分割區。
+
[NOTE]
====
** 在區域配置中使用 SVM FC 邏輯接口，而不是ONTAP系統上的實體 FC 連接埠。
** 對 FC LUN 使用多路徑。


====


.步驟
. 依照適用於ONTAP tools for VMware vSphere文件中的說明，在管理網域上部署ONTAP工具：link:https://docs.netapp.com/us-en/ontap-tools-vmware-vsphere-10/deploy/ontap-tools-deployment.html["在管理域上部署ONTAP工具"^] 。
+
ONTAP tools for VMware vSphere部署為小型單節點，具有核心服務以支援 NFS 和 VMFS 資料儲存庫。

. 依照適用於ONTAP tools for VMware vSphere文件中的說明，使用 vSphere 用戶端介面新增儲存後端：link:https://docs.netapp.com/us-en/ontap-tools-vmware-vsphere-10/configure/add-storage-backend.html["使用 vSphere 用戶端介面定義儲存後端"^] 。
+
新增儲存後端可讓您加入ONTAP叢集。

. 依照適用於ONTAP tools for VMware vSphere文件中的說明在 FC 上設定 VMFS：link:https://docs.netapp.com/us-en/ontap-tools-vmware-vsphere-10/configure/create-datastore.html["在 FC 上配置 VMFS"^] 。
