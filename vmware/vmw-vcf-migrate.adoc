= 了解如何將具有現有資料儲存的 vSphere 環境轉換為 VCF 管理域
:hardbreaks:
:allow-uri-read: 
:nofooter:

[role="lead"]
轉換具有ONTAP上現有光纖通道或 NFS 資料儲存的 vSphere 環境涉及將目前基礎架構整合到現代私有雲架構中。



== 解決方案概述

此解決方案示範了當叢集轉換為 VCF 管理域時，vSphere 中現有的 FC 或 NFS 資料儲存如何成為主要儲存。

此流程受益於ONTAP儲存的穩健性和靈活性，可確保無縫的資料存取和管理。透過轉換過程建立 VCF 管理網域後，管理員可以有效地將其他 vSphere 環境（包括使用 FC 和 NFS 資料儲存的環境）匯入 VCF 生態系統。

這種整合不僅提高了資源利用率，還簡化了私有雲基礎架構的管理，確保了平穩過渡，並最大限度地減少了對現有工作負載的干擾。



== 架構概述

ONTAP工具的架構與 VMware 環境無縫集成，利用包含ONTAP工具服務、vSphere 外掛程式和 REST API 的模組化可擴充框架實現高效的儲存管理、自動化和資料保護。

ONTAP tools for VMware vSphere可以安裝在 HA 或非 HA 配置中。
