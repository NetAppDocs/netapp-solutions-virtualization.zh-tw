[role="lead"]
適用於 Proxmox 的儲存管理器 (SMP) 為使用 NetApp ONTAP 儲存系統的 Proxmox 環境中的虛擬機器提供全面的儲存資源配置和生命週期管理。SMP 使管理員能夠直接管理儲存設備,並透過自動化資源配置、備份、還原和複製功能簡化操作。



== 瞭解 Storage Manager for Proxmox

適用於 Proxmox 的儲存管理器（SMP）為使用 NetApp 儲存系統的 Proxmox 環境中的虛擬機器提供端對端的儲存管理。透過允許管理員直接管理儲存，SMP 簡化了 Proxmox 環境中的儲存和資料管理任務。



=== 主要功能

* *端對端儲存管理*：SMP 為虛擬機器提供完整的生命週期管理，從儲存資源配置到備份和還原。
* *直接儲存管理*：Proxmox 管理員可以直接管理儲存設備，簡化操作並降低複雜性。
* *獨立安裝程式*：SMP 以獨立安裝程式的形式提供，可透過 API 無縫連接到 Proxmox 節點。




=== 功能

SMP 的每個元件都旨在提高儲存管理效率：

* *儲存資源配置*：輕鬆為 Proxmox 主機和關聯的虛擬機器分配和管理儲存資源。
+
** *NFS 儲存資源配置*：設定和配置 ONTAP NFS 儲存設備，以便與您的 Proxmox 環境一起使用。
** *iSCSI 儲存資源配置*：實作具有多路徑的厚 LVM，以實現強大且高效的 iSCSI 儲存管理。


* *備份與還原*：執行可靠的備份與快速還原，以確保資料完整性與可用性。
+
** *當機一致性備份和檔案一致性備份*：透過設定當機一致性備份和檔案一致性備份機制來確保資料完整性。
** *VM 還原作業*：執行虛擬機器還原程序以復原資料並維持系統可用性。


* *儲存效率*：ONTAP 提供一套全面的儲存效率技術，旨在最大限度地減少資料所需的儲存空間。這些技術不僅可以降低儲存消耗，還可以提升系統效能，具體體現在：
+
** *快速 VM 配置*：簡化虛擬機器的配置流程，實現更快部署。
** *更快的資料集複本*:加快資料集的複製速度,以實現高效率的資料管理。






=== 先決條件

在繼續之前，請確保您已完成以下步驟：

* 運行 Proxmox Virtual Environment（PVE）9.x 或更高版本的叢集。
* ONTAP All Flash FAS (AFF) 儲存設備，在 SVM 層級啟用 NFS 和 iSCSI 通訊協定。如需相關指示，請參閱 link:proxmox-ontap-nfs.html["使用 ONTAP 設定 NFS 資料存放區"] 和 link:proxmox-ontap-iscsi.html["使用 ONTAP 儲存設備為 Proxmox VE 9.x 設定 iSCSI"]。
+
[NOTE]
====
對於 iSCSI LUN 組態，應啟用下列功能：

*在所有 Proxmox 主機上安裝多重路徑和 iSCSI ：*

[source, shell]
----
apt update
apt install open-iscsi multipath-tools -y
systemctl enable --now iscsid multipathd
----
*設定 iSCSI 啟動器並透過 iSCSI 連線（在 Proxmox 主機上）：*

建立或更新 `/etc/multipath.conf`：

[source, shell]
----
touch /etc/multipath.conf
----
或使用：

[source, shell]
----
cat << 'EOF' > /etc/multipath.conf
defaults {
   user_friendly_names yes
   find_multipaths yes
}
blacklist {
   devnode "^sd[a]$"
}
EOF
----
====
* Proxmox 主機和 ONTAP 儲存可透過網路從 SMP 主機存取。

== 設定 Proxmox 的 Storage Manager

當儲存設備和與儲存設備節點以及 Proxmox 節點的連線都已正確配置後，設定 SMP 以利用 ONTAP Snapshot 和 FlexClone 功能自動配置和保護虛擬機器。

第一步是探索並新增 ONTAP 儲存端點，然後將目標 Proxmox 節點新增至 SMP。

.開始之前
確保滿足以下先決條件：

* 已安裝 SMP
* SMP 執行個體與 Proxmox 節點和儲存系統具有連線能力
* ONTAP 叢集提供 NFS 或 iSCSI 儲存池
* 使用 DNS 名稱時會自動進行 DNS 解析。否則，請使用 IP 位址進行探索。
* 此環境支援多種版本的 Proxmox Virtual Environment 伺服器


.關於此任務
在 Hypervisor 與儲存設備之間建立連線後，組態設定只需點擊幾下，大約 2 分鐘即可完成。

.步驟
. 登入 SMP。
. 從 SMP 左側導覽列中選擇 *Plan*。
. 點選 *Add Plan* 。

== 配置儲存設備

將磁碟區配置為儲存池（通常稱為資料儲存區）會為虛擬機器及其虛擬磁碟（QCOW2 或 RAW）建立一個邏輯容器。SMP 支援配置資料儲存區，然後將其附加到單一主機或叢集中的所有主機。



=== 配置 NFS 儲存設備

使用 SMP 將 ONTAP 磁碟區配置並掛載為 NFS 資料存放區。

.步驟
. 在 SMP GUI 中，前往 *Provision* 標籤。
. 導覽至 *Add Volume* 。
+
.檢視 Provision 標籤：新增 Volume
[%collapsible]
====
image::proxmox-smp-image13.png[SMP Provision 標籤，帶有 Add Volume 選項]

====
. 在「新磁碟區」精靈的「快速配置」步驟中，選取 *NFS*。
+

NOTE: 預設選擇 NFS。快速配置使用標準部署的預設設定，提供更快速的設定。進階配置提供 HA 和 BCDR 功能的擴充選項，這些功能將在未來版本中推出。

+
.檢視新磁碟區精靈：NFS
[%collapsible]
====
image::proxmox-smp-image14.png[在「快速配置」步驟中選取 NFS 的「新增 Volume 精靈」]

====
. 在 *ONTAP 詳細資料* 部分中，提供下列資訊：
+
.. 從 *Storage Cluster* 下拉式清單中選取 ONTAP 叢集或指定的 Storage Virtual Machine （SVM）
.. 在 *Volume Name* 欄位中提供唯一值
.. 請在 *Size* 欄位中輸入資料存放區磁碟區所需的容量
.. 選擇 ONTAP 儲存資源時，請選擇以下對應的選項：
+
*** Aggregate 名稱
*** 儲存虛擬機器（SVM）
*** 用於資料存放區存取的首選 LIF


.. 選擇現有的 *Export Policy* 或根據需要點擊 (+) 建立自訂策略
+
.檢視 ONTAP 詳細資料：NFS
[%collapsible]
====
image::proxmox-smp-image15.png[用於 NFS 磁碟區配置的 ONTAP 詳細資訊區段]

====


. 在 *Proxmox 詳細資料* 部分中，提供下列資訊：
+
.. 在 *Proxmox* 欄位中，選擇目標叢集，然後選取所有需要存取資料存放區的節點
.. 在 *NFS Version* 欄位中、選取偏好的 NFS 傳輸協定（NFS v3 或 NFS v4.1）
.. 在 *Content* 區段中、選取要為此資料存放區啟用的內容類型：
+
*** 磁碟映像
*** ISO 映像
*** 容器範本
*** 備份
*** 片段
*** 匯入
*** 或選擇 *All* 內容類型


.. （選用）如果需要多個 TCP 連線，請啟動 *nconnect* 選項
.. 確保已勾選 *Mount the datastore*，以便將資料儲存區掛載到選定的 Proxmox 節點上
+
.檢視 Proxmox 詳細資料：NFS

. 點選磁碟區旁的三個豎點（⋮）。
. 從內容功能表中選取 *Edit* 以開啟磁碟區設定。
+
.檢視編輯磁碟區對話方塊
[%collapsible]
====
image::proxmox-smp-image18b.png[顯示磁碟區組態選項的「編輯磁碟區」對話方塊]

====
. 依需求修改磁碟區設定：
+
** *調整磁碟區大小*：調整磁碟區的儲存容量
+

NOTE: 調整大小僅適用於 NFS 磁碟區。

+

NOTE: 這需要在叢集上安裝 SnapLock 授權，並且必須初始化叢集的合規時鐘。

. 按一下 *Save* 以套用變更。




=== 配置 iSCSI 儲存設備

使用 SMP 為虛擬機器設定 iSCSI 後端。

.步驟
. 在 SMP GUI 中，前往 *Provision* 標籤。
. 導覽至 *Add Volume* 。
. 在「新磁碟區」精靈的「快速配置」步驟中，選取 *iSCSI* 。
+
.檢視新磁碟區精靈：iSCSI
[%collapsible]
====
image::proxmox-smp-image19.png[在「快速配置」步驟中選取 iSCSI 的「新增 Volume 精靈」]

+
.. 從 *Storage Cluster* 下拉式清單中選擇適當的 ONTAP 叢集或指定的 Storage Virtual Machine（SVM）
.. 選取下列選項：
+
*** Aggregate 名稱
*** 儲存虛擬機器（SVM）


.. 請在 *LUN Name* 欄位中輸入唯一值
.. 在 *LUN 數量* 欄位中提供所需的 LUN 數量（預設值為 1；可並行建立相同大小的其他 LUN）
.. 在 *每個 LUN 的容量* 欄位中輸入每個 LUN 所需的大小
.. 為 iSCSI 存取控制選擇一個 *Initiator Group*（igroup）。可以選擇現有的 igroup，也可以建立一個新的 igroup。
+
.檢視 ONTAP 詳細資料：iSCSI LUN 組態
[%collapsible]
====
image::proxmox-smp-image20.png[iSCSI ONTAP 詳細資料部分顯示 LUN 設定選項]

.. （選用）如果需要 VLAN 分段，請選擇所需的 *Storage VLAN*
.. 在 *Content* 區段中，選取此資料存放區支援的內容類型：

*** 磁碟映像
*** 容器
*** 或選擇 *All* 內容類型


.. 請確保選取 *Mount the datastore* ，以便所選的 Proxmox 節點均可存取該儲存。
.. （選用）如果儲存資源池將由多個節點同時存取，請啟用 *Shared*

=== NConnect 行為

在 Proxmox VE 中使用 NFS 儲存時，請注意 Linux NFS 用戶端（例如 Proxmox VE 的基礎作業系統 Debian 所使用的客戶端）的特定行為。Linux 核心可能會重複使用首次在節點上掛載 NFS 匯出時所使用的掛載參數。如果使用不同的掛載選項再次掛載同一個匯出，則這些新選項可能不會生效。

最佳實務做法是建立並掛載個別的匯出，並使用不同的 NFS 伺服器 IP 位址，例如不同的資料 LIF。SMP 會自動偵測現有連線及其相關的 nconnect 值，並將適當的 nconnect 值套用至任何新連線。



== 保護虛擬機器

當虛擬機器在 ONTAP 型 NFS 資料存放區上執行時，最簡化的保護方法是讓 Proxmox 建立檔案系統一致性，而 ONTAP 在儲存層擷取實際的備份點。ONTAP Snapshot 可提供資料存放區磁碟區的快速、節省空間的時間點複本，並可在稍後用於還原整個資料存放區、個別檔案或 VM 內容。

SMP 提供了一個簡單的協調器，用於實現崩潰一致性和虛擬機器一致性的資料保護和複製管理。該軟體透過建立可應用於多個作業的備份策略，簡化了備份、還原和複製的生命週期。這些策略可以定義排程、保留、複寫和其他功能。它們允許選擇性地使用檔案一致性快照；這種方法利用了 QEMU 在建立 ONTAP 快照之前暫停 I/O 的功能。

SMP 允許保護整個資料存放區，其中一個 ONTAP NFS 磁碟區包含多個 VM、可能跨多個資料存放區的 VM 群組，或單一 VM 層級。

=== 還原工作流程詳細資料

還原工作流程因底層儲存類型而異，主要包括以下幾個關鍵步驟：

. *定義還原範圍*：選擇保護目標是資料存放區（ONTAP 磁碟區）、資源群組（多個磁碟區或 VM）還是個別 VM。
. *執行還原作業*：VM 還原程序會因儲存傳輸協定而異：
+
** *基於 NFS 的還原*：在 NFS 上還原磁碟區或資料存放區時，SMP 會在目標磁碟區上啟動磁碟區快照還原作業。對於託管在 NFS 磁碟區上的 VM，SMP 會驗證還原參數，包括 VM ID、快照名稱、站台 ID 和儲存類型。如果 VM 正在執行，SMP 會輪詢其狀態並發出停止命令來停止它。然後，系統會使用檔案層級還原作業從快照還原 VM 磁碟。
** *基於 iSCSI 的還原*：對於 iSCSI 資料存放區或 LUN、SMP 會從快照還原 LUN、驗證還原的裝置、在 Proxmox 節點上執行 iSCSI 重新掃描、並確保裝置可存取。對於駐留在 iSCSI 儲存設備上的 VM、SMP 會複製快照 LUN、將複製的 LUN 對應到 Proxmox 主機、並執行 iSCSI 探索以匯入它們。然後會新增還原的儲存設備、並更新 VM 組態。VM 磁碟會移回其原始儲存設備、並追蹤每個 VM 的進度。此程序最後會進行清理活動、包括移除暫存儲存設備和刪除複製的 LUN。

== 總結

Storage Manager for Proxmox 提供了一種簡化且統一的方式來管理 Proxmox 環境中的保護、複製和復原作業。透過與 ONTAP 直接整合，該解決方案將企業級儲存的效率、可靠性和效能融入日常 VM 工作流程中。ONTAP 的空間高效 Snapshot、快速複製功能和內建複寫功能可提供一致且可預測的資料保護，而 SMP 則將這些功能簡化到一個直觀、以任務為中心的介面中。
